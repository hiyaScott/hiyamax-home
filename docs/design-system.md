## 一、品牌命名解释

**HiyaMax** = **Hiya** + **Max**

- **Hiya**: 英式俚语中的问候语，轻松、友好、带有创作者的个人印记
- **Max**: 最大化、极致、边界探索者
- **组合含义**: "极致的问候" —— 既是向世界的自我介绍，也是对创作极限的不断突破

**标语**: *Player & Creator*（玩家与创作者的双重身份）

## 二、页面层级定义

### 一级页面（主页面 / Landing Page）

**定义**: 用户进入网站的入口，展示品牌核心身份和全局导航

**实例**: `about` / 首页 (`/`)

**特征**:
- 大型 Hero 区域（双头像设计）
- 品牌标语 "Player & Creator"
- 全局导航入口（games | books | artworks | hiyaMax™️）
- 社交链接
- 最新作品预览（3-4 个精选）

**设计特点**:
- 视觉冲击力最强
- 动画效果最丰富（头像随鼠标移动、网格背景等）
- 信息密度中等，强调品牌感知

---

### 二级页面（分类页 / Category Page）

**定义**: 展示某一内容类别的集合页，承上启下

**实例**:

| 页面 | 路径 | 内容 |
|------|------|------|
| Games | `/games/` | 游戏作品集 |
| Books | `/books/` | 书籍/阅读 |
| Artworks | `/artworks/` | 艺术创作 |
| hiyaMax™️ | `/hiyamax/` | 品牌/IP 项目 |

**核心设计规范**:

1. **布局系统**: Flex 纵向布局
   - `body`: `display: flex; flex-direction: column;`
   - `page-container`: `flex: 1;`（撑满中间区域）
   - `footer`: `margin-top: auto;`（被推到底部）

2. **背景处理**:
   - 网格背景: `position: fixed; z-index: 0`（全屏固定）
   - 内容区域: `background: transparent`（透出网格）
   - 页脚区域: `background: #fff`（白色覆盖网格）

3. **卡片设计**:
   - 上半部分: 深色渐变背景 (`#2a2a3a → #1a1a2e`)
   - 下半部分: 半透明白色 (`rgba(255,255,255,0.8)`)
   - 卡片间距: 24px

**设计特点**:
- 大标题背景（如 "GAMES" 固定文字）
- 交互式网格背景（随鼠标移动）
- 响应式网格（3→2→1 列）
- 每页 9 张卡片
- 移除 Scroll 指示器

---

### 三级页面（详情页 / Detail Page）

**定义**: 单个项目的完整展示页，内容最详尽

**实例**:

| 父级 | 三级页面 | 路径示例 |
|------|---------|---------|
| Games | 具体游戏详情 | `/games/blade-chronicles/` |
| Books | 具体书籍详情 | `/books/pixel-world-explorer/` |
| Artworks | 具体作品详情 | `/artworks/project-name/` |

**核心设计规范**:

1. **整体主题**: 浅色主题
   - 背景: `#f8fafc` (浅灰白)
   - 文字: `#333` (深灰)

2. **面包屑导航**:
   - 背景: `#fff` (纯白)
   - 边框: `1px solid #eee`
   - 链接颜色: `#666` → hover `#000`

3. **内容区域**:
   - 背景: `#f8fafc`
   - 文字: `#333`
   - 标题: `#111`

4. **页脚**: 与二级页面统一
   - 背景: `#fff`
   - padding: `30px 40px 60px`
   - border-top: `1px solid #eee`

**统一模板结构**:

```
┌─────────────────────────────────────┐
│  [返回 Games]                       │  ← 面包屑导航
├─────────────────────────────────────┤
│                                     │
│  [项目封面图 / 视频 / 交互演示]      │  ← Hero 媒体区
│  尺寸: 16:9 或 4:3                   │
│                                     │
├─────────────────────────────────────┤
│  [项目标题]                         │
│  [副标题/一句话简介]                │
├─────────────────────────────────────┤
│  类别: [标签] | 年份: 2024          │  ← 元信息
│  平台: [平台图标]                    │
├─────────────────────────────────────┤
│                                     │
│  [详细描述]                         │  ← 内容区
│  技术栈/工具                        │
│  创作背景                           │
│                                     │
├─────────────────────────────────────┤
│  [图库/截图网格]  [视频嵌入]        │  ← 媒体展示区
│                                     │
├─────────────────────────────────────┤
│  [← 上一个项目]    [下一个项目 →]    │  ← 项目导航
└─────────────────────────────────────┘
```

**模板组件规范**:

| 组件 | 必需 | 说明 |
|------|------|------|
| 面包屑 | ✅ | 返回父级分类页 |
| Hero 媒体 | ✅ | 封面图/视频/GIF，自适应容器 |
| 标题 | ✅ | 项目全称 |
| 副标题 | 可选 | 一句话卖点 |
| 元信息栏 | ✅ | 类别标签、年份、平台图标 |
| 内容区 | ✅ | 详细描述，支持 Markdown |
| 技术栈 | 可选 | 图标 + 文字标签 |
| 媒体网格 | 可选 | 2-3 列图片/视频 |
| 外部链接 | 可选 | 试玩、下载、GitHub |
| 项目导航 | ✅ | 上一个/下一个项目卡片 |

## 三、设计哲学

**核心理念**: 一体两面
- 既是内容的**消费者**（Player），也是内容的**创造者**（Creator）
- 这种双重性体现在视觉、交互和内容各个层面

**设计关键词**: 简洁、克制、现代、技术感、留白

## 四、视觉规范

### 色彩系统

#### 全局色彩

| 变量名 | 值 | 用途 |
|--------|-----|------|
| `--bg-primary` | `#ffffff` | 主背景（页脚、面包屑）|
| `--bg-secondary` | `#f8fafc` | 次级背景（三级页面内容区）|
| `--text-primary` | `#0f172a` | 主文字 |
| `--text-secondary` | `#666666` | 次级文字 |
| `--border-light` | `#eeeeee` | 边框 |

#### 二级页面色彩

| 元素 | 背景 | 文字 |
|------|------|------|
| 网格背景 | `rgba(220,220,220,0.6)` | - |
| 卡片上半 | 渐变 `#2a2a3a → #1a1a2e` | 白色 |
| 卡片下半 | `rgba(255,255,255,0.8)` | `#111` |
| 页脚 | `#fff` | `#666` |

#### 三级页面色彩

| 元素 | 背景 | 文字 |
|------|------|------|
| 面包屑 | `#fff` | `#666` → hover `#000` |
| 内容区 | `#f8fafc` | `#333` |
| 页脚 | `#fff` | `#666` |
| 代码块 | `#1e293b` | `#e2e8f0` |

### 字体系统

- **主字体**: Inter (Google Fonts)
- **字重范围**: 200 (Ultra Light) - 700 (Bold)
- **使用原则**: 
  - 导航: 字重 200，小写，营造精致感
  - 标题: 字重 800，大写，视觉冲击
  - 正文: 字重 400-600，可读性优先

### 布局系统

| 断点 | 宽度 | 网格列数 |
|------|------|---------|
| Desktop | 1200px | 3列 |
| Tablet (1024px) | 100% | 2列 |
| Mobile (640px) | 100% | 1列 |

**间距规范**:
- 容器边距: 40px (桌面) → 30px (平板) → 20px (手机)
- 卡片间距: 24px (桌面) → 16px (手机)
- 组件内边距: 20px 标准

## 五、组件规范

### 游戏卡片 (Game Card) - 二级页面

```
┌─────────────────────────┐
│  [emoji]    [类型标签]   │  ← 深色背景区域 (渐变: #2a2a3a → #1a1a2e)
│            [平台标签]    │
├─────────────────────────┤
│  游戏名称                 │  ← 半透明白色背景区域 (rgba(255,255,255,0.8))
│  类别 · 年份             │
└─────────────────────────┘
```

**规格**:
- 卡片圆角: 16px
- 阴影: `0 4px 12px rgba(0,0,0,0.08)`
- Hover: `translateY(-4px)` + 阴影加深
- 上半高度: 160px

### 页脚 (Footer) - 全站统一

**样式规范** (适用于所有页面):
- 背景: `#fff`
- 上边框: `1px solid #eee`
- padding: `30px 40px 60px`（底部加大，延伸到页面底部）
- 链接颜色: `#666` → hover `#000`

**技术实现** (二级/三级页面):
```css
body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.page-container {
  flex: 1;
}

.footer {
  margin-top: auto;
  background: #fff;
}
```

### 导航栏 (Header)

- 固定定位，z-index: 100
- 高度: 70px
- Logo + 主导航 (about | games | books | artworks | hiyaMax™️)
- 社交图标: 微信、小红书、Instagram、Facebook

## 六、动效规范

### 网格背景 (Grid Background)

- 单元格大小: 30px × 30px，间距 4px
- 颜色: `rgba(220, 220, 220, 0.6)`
- 交互: 鼠标靠近时单元格旋转 + 放大
- 影响半径: 400px
- 动画缓动: `0.1` 平滑系数

### 页面过渡

- 三级页面进入: 淡入 + 轻微上移 (200ms)
- 卡片 Hover: 0.3s ease-out

---

## 七、技术规范

### 布局技术

**Flex 纵向布局** (二级/三级页面):
```css
html {
  height: 100%;
}

body {
  min-height: 100vh;
  display: flex;
  flex-direction: column;
}

.page-container {
  flex: 1;
  position: relative;
  z-index: 2;
}

.footer {
  margin-top: auto;
  position: relative;
  z-index: 3;
}
```

**层级管理**:
| 元素 | z-index |
|------|---------|
| 网格背景 (canvas) | 0 |
| GAMES 标题 | 1 |
| 内容区域 | 2 |
| 页脚 | 3 |

### 框架与结构

- **框架**: Jekyll (静态站点生成)
- **布局继承**:
  - 一级页面: `default` → `header` + `content` + `footer`
  - 二级页面: `default` → `games-list` (或其他分类)
  - 三级页面: `default` → `game` (或其他详情)
- **样式**: 内联 `<style>` (单页面) 或外部 CSS
- **脚本**: Vanilla JS，无框架依赖
- **图标**: 内联 SVG
