# HiyaMax 网站设计规范与命名解释

## 一、品牌命名解释

**HiyaMax** = **Hiya** + **Max**

- **Hiya**: 英式俚语中的问候语，轻松、友好、带有创作者的个人印记
- **Max**: 最大化、极致、边界探索者
- **组合含义**: "极致的问候" —— 既是向世界的自我介绍，也是对创作极限的不断突破

**标语**: *Player & Creator*（玩家与创作者的双重身份）

---

## 二、页面层级定义

### 一级页面（主页面 / Landing Page）

**定义**: 用户进入网站的入口，展示品牌核心身份和全局导航

**实例**: `about` / 首页 (`/`)

**特征**:
- 大型 Hero 区域（双头像设计）
- 品牌标语 "Player & Creator"
- 全局导航入口（games | books | artworks | hiyaMax™️）
- 社交链接
- 最新作品预览（3-4 个精选）

**设计特点**:
- 视觉冲击力最强
- 动画效果最丰富（头像随鼠标移动、网格背景等）
- 信息密度中等，强调品牌感知

---

### 二级页面（分类页 / Category Page）

**定义**: 展示某一内容类别的集合页，承上启下

**实例**:

| 页面 | 路径 | 内容 |
|------|------|------|
| Games | `/games/` | 游戏作品集 |
| Books | `/books/` | 书籍/阅读 |
| Artworks | `/artworks/` | 艺术创作 |
| hiyaMax™️ | `/hiyamax/` | 品牌/IP 项目 |

**特征**:
- 大标题背景（如 "GAMES" 固定文字）
- 网格布局的内容卡片
- 分页或筛选功能
- 统一的卡片设计系统
- 返回主页的清晰路径

**设计特点**:
- 交互式网格背景（随鼠标移动）
- 卡片 Hover 效果
- 响应式网格（3→2→1 列）
- 统一的卡片结构（图片区 + 信息区）

---

### 三级页面（详情页 / Detail Page）

**定义**: 单个项目的完整展示页，内容最详尽

**实例**:

| 父级 | 三级页面 | 路径示例 |
|------|---------|---------|
| Games | 具体游戏详情 | `/games/blade-chronicles/` |
| Books | 具体书籍详情 | `/books/pixel-world-explorer/` |
| Artworks | 具体作品详情 | `/artworks/project-name/` |

**统一模板结构**:

```
┌─────────────────────────────────────┐
│  [返回 Games]                       │  ← 面包屑导航
├─────────────────────────────────────┤
│                                     │
│  [项目封面图 / 视频 / 交互演示]      │  ← Hero 媒体区
│  尺寸: 16:9 或 4:3                   │
│                                     │
├─────────────────────────────────────┤
│  [项目标题]                         │
│  [副标题/一句话简介]                │
├─────────────────────────────────────┤
│  类别: [标签] | 年份: 2024          │  ← 元信息
│  平台: [平台图标]                    │
├─────────────────────────────────────┤
│                                     │
│  [详细描述]                         │  ← 内容区
│  技术栈/工具                        │
│  创作背景                           │
│                                     │
├─────────────────────────────────────┤
│  [图库/截图网格]  [视频嵌入]        │  ← 媒体展示区
│                                     │
├─────────────────────────────────────┤
│  [← 上一个项目]    [下一个项目 →]    │  ← 项目导航
└─────────────────────────────────────┘
```

**模板组件规范**:

| 组件 | 必需 | 说明 |
|------|------|------|
| 面包屑 | ✅ | 返回父级分类页 |
| Hero 媒体 | ✅ | 封面图/视频/GIF，自适应容器 |
| 标题 | ✅ | 项目全称 |
| 副标题 | 可选 | 一句话卖点 |
| 元信息栏 | ✅ | 类别标签、年份、平台图标 |
| 内容区 | ✅ | 详细描述，支持 Markdown |
| 技术栈 | 可选 | 图标 + 文字标签 |
| 媒体网格 | 可选 | 2-3 列图片/视频 |
| 外部链接 | 可选 | 试玩、下载、GitHub |
| 项目导航 | ✅ | 上一个/下一个项目卡片 |

**设计特点**:
- 沉浸式布局，媒体优先
- 信息层级：标题 → 元信息 → 内容 → 媒体 → 导航
- 统一的返回路径
- 项目间顺畅跳转

## 三、设计哲学

**核心理念**: 一体两面
- 既是内容的**消费者**（Player），也是内容的**创造者**（Creator）
- 这种双重性体现在视觉、交互和内容各个层面

**设计关键词**: 简洁、克制、现代、技术感、留白

## 四、视觉规范

### 色彩系统

| 变量名 | 值 | 用途 |
|--------|-----|------|
| `--bg-primary` | `#ffffff` | 主背景 |
| `--bg-secondary` | `#f8fafc` | 次级背景/卡片 |
| `--text-primary` | `#0f172a` | 主文字 |
| `--text-secondary` | `#666666` | 次级文字 |
| `--accent-dark` | `#1a1a2e` | 深色强调（卡片头部）|
| `--border-light` | `#eeeeee` | 边框 |

### 字体系统

- **主字体**: Inter (Google Fonts)
- **字重范围**: 200 (Ultra Light) - 700 (Bold)
- **使用原则**: 
  - 导航: 字重 200，小写，营造精致感
  - 标题: 字重 800，大写，视觉冲击
  - 正文: 字重 400-600，可读性优先

### 布局系统

| 断点 | 宽度 | 网格列数 |
|------|------|---------|
| Desktop | 1200px | 3列 |
| Tablet (1024px) | 100% | 2列 |
| Mobile (640px) | 100% | 1列 |

**间距规范**:
- 容器边距: 40px (桌面) → 30px (平板) → 20px (手机)
- 卡片间距: 24px (桌面) → 16px (手机)
- 组件内边距: 20px 标准

## 五、组件规范

### 游戏卡片 (Game Card) - 二级页面

```
┌─────────────────────────┐
│  [emoji]    [类型标签]   │  ← 深色背景区域 (渐变: #2a2a3a → #1a1a2e)
│            [平台标签]    │
├─────────────────────────┤
│  游戏名称                 │  ← 白色背景区域
│  类别 · 年份             │
└─────────────────────────┘
```

**交互**:
- Hover: `translateY(-4px)` + 阴影加深
- 过渡: `0.3s ease-out`

### 项目卡片 (Project Card) - 三级页面导航

```
┌──────────────────┐
│ [缩略图/emoji]   │
├──────────────────┤
│ 项目名称         │
│ 简短描述        │
└──────────────────┘
```

### 导航栏 (Header)

- 固定定位，z-index: 100
- 高度: 70px
- Logo + 主导航 (about | games | books | artworks | hiyaMax™️)
- 社交图标: 微信、小红书、Instagram、Facebook

### 底栏 (Footer)

- 与导航栏内容呼应
- 版权信息 + 版本号

## 六、动效规范

### 网格背景 (Grid Background)

- 单元格大小: 30px × 30px，间距 4px
- 颜色: `rgba(220, 220, 220, 0.6)`
- 交互: 鼠标靠近时单元格旋转 + 放大
- 影响半径: 400px
- 动画缓动: `0.1` 平滑系数

### 滚动指示器

- 位置: 内容区底部居中
- 动画: 上下浮动 (2s 循环)
- 提示文字: "SCROLL"

### 页面过渡

- 三级页面进入: 淡入 + 轻微上移 (200ms)
- 卡片 Hover: 0.3s ease-out

---

## 七、技术规范

- **框架**: Jekyll (静态站点生成)
- **布局继承**: 
  - 一级/二级: `default` → `header` + `content` + `footer`
  - 三级: `default` 或专用 `detail` 布局
- **样式**: 内联 `<style>` (单页面) 或外部 CSS
- **脚本**: Vanilla JS，无框架依赖
- **图标**: 内联 SVG

## 八、内容规范

### 游戏数据字段

```yaml
title: 游戏名称
emoji: 🎮
category: 类别
platform: 平台
year: 年份
description: 简介
cover_image: 封面图路径
detail_page: true  # 是否有三级详情页
```

### 分页规则

- 每页显示: 6 个游戏卡片
- 占位卡片: 9 个预设（当真实游戏不足时显示）

### 三级页面数据字段

```yaml
title: 项目全称
subtitle: 一句话卖点
category: 所属分类
year: 2024
platform: [平台列表]
cover: 封面媒体路径
description: |
  详细描述，支持多行
tech_stack: [技术标签]
media:
  - type: image
    src: 图片路径
    caption: 描述
  - type: video
    src: 视频链接
links:
  play: 试玩链接
  download: 下载链接
  github: 源码链接
prev_project: 上一个项目ID
next_project: 下一个项目ID
```
